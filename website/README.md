## Run it

```bash
npm install
npm run dev          # http://localhost:4321
npm run build        # static output in dist/
npm run preview      # preview the production build locally
```

`predev` and `prebuild` automatically run `scripts/sync-legal.mjs`, which copies the canonical legal markdown from the iOS repo into `src/content/legal/`. Don't hand-edit the files in `src/content/legal/` ‚Äî edit the originals at the repo root.

## Before launching

A few one-time things to update before flipping the DNS:

1. **Domain** ‚Äî `astro.config.mjs ‚Üí site` and `src/lib/site.ts ‚Üí SITE.url`. Both should point at your real production domain.
2. **Aggregate rating** ‚Äî `src/lib/site.ts ‚Üí SITE.rating`. Keep in sync with App Store Connect numbers, or set to `null` to omit the `aggregateRating` JSON-LD.
3. **Hero rating numbers** in copy ‚Äî search `4.8` if you don't want to surface a hard-coded rating before reviews land.

The App Store ID (`6763663209`) and Smart App Banner are already wired through `SITE.appStoreId` / `SITE.appStoreUrl` and emit on every page.

## Deploy

The site is plain static HTML ‚Äî every host that serves files works:

- **Cloudflare Pages** ‚Äî `npm run build`, point at `dist/`. Recommended; free, fast, on the same network as `speed.cloudflare.com` if you ever want to dogfood your own speed test against the marketing site.
- **Vercel** ‚Äî Astro adapter is auto-detected.
- **Netlify** ‚Äî same.
- **GitHub Pages** ‚Äî `dist/` is publishable, but you'll need to override the `site:` URL or set up a custom domain.
- **S3 + CloudFront** ‚Äî drop `dist/` on S3, point CloudFront at it, set `index.html` as the default root + `404.html` as the error doc.

After deploy, submit the sitemap to:

- [Google Search Console](https://search.google.com/search-console) ‚Äî `https://wifibuddy.app/sitemap-index.xml`
- [Bing Webmaster Tools](https://www.bing.com/webmasters) ‚Äî same URL

## Adding a blog post

Drop a new `.mdx` file in `src/content/blog/` with frontmatter:

```mdx
---
title: "Your post title"
description: "150-160 character search-snippet description."
pubDate: 2026-05-01
keywords: ["primary keyword", "secondary keyword"]
---

Markdown body. MDX components allowed.
```

The post is automatically picked up by:

- `/blog/index.astro` (sorted by `pubDate` descending, drafts hidden)
- `/blog/[...slug].astro` (renders the post body)
- `@astrojs/sitemap` (added to the sitemap)
- `Article` JSON-LD (auto-generated from the frontmatter)

The slug is the filename minus `.mdx`. Set `draft: true` in frontmatter to keep a post out of the index but still publish at its slug for review.

## Updating the legal docs

Don't edit `src/content/legal/*.md` directly. Edit the canonical files:

- `<repo-root>/PrivacyPolicy.md`
- `<repo-root>/SignalStrengthPainter/TermsOfUse.md`

`scripts/sync-legal.mjs` re-copies them on every `npm run dev` and `npm run build`. Both the iOS app's in-app `LegalDocumentView` and the public `/privacy` and `/terms` URLs serve the same text ‚Äî App Store reviewers can audit one against the other.

## Brand tokens

The traffic-light + dark-gray palette mirrors `AppLogoView.swift` and `scripts/generate_app_icon.py` in the parent repo. The constants live in `src/styles/global.css` under the `@theme { ... }` block:

- `--color-bg` / `--color-bg-card` / `--color-border` ‚Äî dark surface stack
- `--color-signal-green` `#4DD966` ‚Äî outer Wi-Fi arc (excellent)
- `--color-signal-yellow` `#FAD138` ‚Äî middle Wi-Fi arc (fair)
- `--color-signal-orange` `#FA8538` ‚Äî inner Wi-Fi arc (poor)
- `--color-signal-red` `#FA5252` ‚Äî center dot (dead zone)
- `--color-signal-blue` `#3B82F6` ‚Äî interactive accent

If you change the iOS app's brand palette, update these tokens too so the site stays consistent.
